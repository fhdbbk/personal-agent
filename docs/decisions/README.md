# Architecture Decision Records (ADRs)

Numbered, immutable records of significant design decisions. Format: `NNNN-kebab-title.md`.

Template per ADR:
- **Status**: Proposed / Accepted / Superseded by NNNN
- **Context**: the situation that forced a decision
- **Decision**: what we picked
- **Consequences**: what gets easier, what gets harder
- **Alternatives considered**: what we rejected and why

When superseding an old ADR, write a new one and update the old one's status — never edit the old decision in place.
